Кодирование звука в формат MP3


    Мало кто сейчас уже не сталкивался с mp3, или хотя бы не слышал о них. Думаю, что нет нужды объяснять преимущества mp3 перед обычными PCM wav файлами, кроме того, мы уже поднимали эту тему в одной из наших статей. Сейчас мы поговорим о программах, которые смогут вам помочь в создании собственной mp3 коллекции.
    Получение mp3 с треков CD-ROM.
    Существует не так уж много способов получить mp3 файлы c СD треков. В большинстве случаев берется wav файл, который при помощи специальной программы - конвертора, преобразуется в mp3.
    При записи с CD-Rom следует пользоваться программами, которые пишут с CD, минуя аудиоплату, таким образом, качество записи теряется не так сильно, как если бы вы писали с CD через аудиоплату. Программы этого класса называются CD-рипперы (или грабберы). Обычно CD-ripper делает из CD треков PCM wav файлы, а затем кодирует полученные wav в mp3 (некоторые рипперы могут создавать mp3 непосредственно с CD трека, без промежуточного wav). Некоторые CD-ripper'ы не имеют mp3 кодека, а поэтому, после получения wav, может понадобиться программа-конвертор в mp3. Большинство рипперов могут нормализовать wav полученный с CD перед кодированием в mp3, эта операция будет полезной в том случае если музыка на CD записана с различной максимальной громкостью (амплитудой), в этом случае все композиции будут приведены к одной, общей для всех, максимальной громкости. Некоторые (далеко не все) способны кодировать в mp3 с переменным битрэйтом. Режим кодирования в mp3 с переменным битрэйтом (VBR - Variable Bit Rate) дает существенный выигрыш в качестве звука, однако размер файла получается непредсказуемым. В режиме VBR программа будет выбирать наилучший битрэйт для каждого отдельно взятого фрейма, исходя из текущего звукового наполнения.
    Большинство программ описанных здесь используют ASPI. ASPI - не так давно появившийся способ чтения данных с аудио CD. ASPI был разработан для устройств SCSI, но он неплохо работает и IDE (когда используется набор команд ATAPI). Если вы пользуетесь Windows 95 или Windows 98, то у вас уже имеется встроенная поддержка ASPI, к Windows NT это, к сожалению, не относится, а для того чтобы все это заработало на NT, вам придется запустить файл ASPI32.exe который можно загрузить с ЭТОЙ страницы.
    
    CDex
    
    По всем признакам лучший риппер. Самое главное - совершенно бесплатный. Работает медленно, но верно. Может работать просто как конвертер. Содержит море опций, в то же самое время не производит впечатления перегруженности, всеми опциямипользоваться удобно. Оригинальная система раскладывания файлов по каталогам: по умолчанию обзывает файл по имени композиции, создает каталог с именем исполнителя в котором будут подкаталоги с названиями альбомов. Причем названия исполнителя, произведения и альбома может брать как с cdplayer.ini, так и с сервера CDDB. Может использовать VBR (при чем VBR настраивается намного гибче, чем в любой другой программе - имеется 10, уровней самый качественный - 0, если не хотите использовать VBR - none). Способен прописывать ID3 tag автоматически, исходя из текущей информации о композиции. Можно настроить так - что бы не забирал все процессорное время. Один из немногих конвертеров способный кодировать c CD непосредственно в mp3 (минуя промежуточный wav). Загрузить последнюю версию можно с Homepage
    
    WinDac32
    
    Еще один риппер с простым и аккуратным интерфейсом. Родом из германии. Гораздо меньше возможностей, чем у CDex. Способен риповать в wav и другие форматы установленные в Windows. В той конфигурации, в которой поставляется, не умеет создавать mp3 (хотя если в вашем Windows встроенный mp3 кодек - можете воспользоваться им). Специальное дополнение можно будет заказать потом. Иногда рипует с присвистываниями, что очень раздражает, для того чтобы этот эффект не появлялся приходиться менять установки по умолчанию. Кроме того, отсутствие встроенного help делает программу еще менее привлекательной. Если кто заинтересовался - скачать можно на Homepage WinDAC
